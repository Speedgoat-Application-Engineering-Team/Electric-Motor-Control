Description

Start your model-based design journey right from your desktop by creating a model for the actual plant (hardware) which captures most of the important features of the motor drive, consisting of the brushless DC motor, power stage drive, current sensor and encoder. You can do this by combining MathWorks physical modelling tools such as Simulink, Simscape Electrical or Powertrain Blockset together with powerful tools that let you collect data directly from hardware and calculating motor parameters.
Once the plant model is created and fine-tuned, you can proceed with the design and fine tune of your controller, for instance using Simulink Design Optimization, so that it meets specific performance requirements while respecting hardware limit.
This step is called Model-in-Loop (MIL) as you are testing the controller logic on the simulated model of the plant. This provides you with a god foundation to test your control algorithm, including supervisory logic and fault detection, using closed-loop desktop simulations.

Learn how to

  • Create an accurate motor model for the brushless DC motor with different methods that let you collect data directly from hardware and calculating motor parameters
  • Automatically compute initial PI controller gains for speed and current loops based on motor and inverter parameters
  • Test your control algorithm, including supervisory logic and fault detection, using closed-loop desktop simulations

Requirements

The exercises Characterize Motor Parameters (Inertia) using Curve Fitting and Characterize Motor Inertia and Friction using Simulink Design Optimization, require you to collect data directly from hardware and calculating motor parameters using Speedgoat target hardware and Speedgoat Electric Motor Control kit. For further information about this setup, please refer to:
  • Getting started with Speedgoat Electric Motor Control Kit and Setup and Calibrate Electric Motor Kit for Rapid Control Prototyping if using the the newest Speedgoat Electric Motor Control kit or,
  • IO397 Demo Kits - User Manual, if you are using the old kit version

Exercises

Test Supervisory Logic, Velocity & Current Closed-Loop Control

  • Combine with physical model and state-logic based scheduling algorithm for full system simulation
  • Evaluate controller performance using desktop simulations (Model-In-Loop Testing)
  • Add formal test cases to simulate difference scenario using Signal Editor
  • Visualize and compare simulation results in Simulation Data Inspector

Characterize Motor Parameters (Inertia) using Curve Fitting

  • Perform Coastdown test to generate data for rotor speed
  • Use Simulation Data Inspector to visual data
  • Use Curve Fitting Tool to calculate rotor inertia, use custom equation to fit curve
  • Update parameter values to simulate and compare the response

Characterize Motor Inertia and Friction using Simulink Design Optimization

  • Perform Coastdown test to generate data for rotor speed
  • Use test data in SPETOOL to compare with simulation result and generate cost function
  • Use optimization technique to estimate Disc Inertia and Viscous Friction Coefficient
  • Update parameter values in model

Tune Closed-Loop Velocity Controller Parameters

  • Use response optimization technique to tune control parameters and test them in real-time on Speedgoat Target Machine
